User journeys map the complete experience of a user interacting with your product. They reveal touchpoints, pain points, and opportunities for improvement.

## What is a User Journey

A user journey is a visual representation of the step-by-step experience a user has while achieving a goal or completing a task.

### Why User Journeys Matter

✅ **See Big Picture** - Understand complete experience
✅ **Identify Gaps** - Find where experience breaks down
✅ **Reveal Touchpoints** - All ways users interact
✅ **Spot Pain Points** - Where users struggle
✅ **Find Opportunities** - Where to improve
✅ **Align Teams** - Everyone sees same picture
✅ **Prioritize Fixes** - Focus on highest-impact areas

### Journey vs. Funnel

```
❌ Funnel (Limited View)
Step 1 → Step 2 → Step 3 → Step 4
Only shows the path

✅ Journey (Complete View)
Step 1 → Step 2 → Step 3 → Step 4
+ Emotions at each step
+ Pain points
+ What they think
+ What they need
+ Opportunities
```

## Journey Map Components

### Essential Elements

**User/Persona**
- Which persona is this for?
- Their goal
- Their context (when/where)

**Phases/Stages**
- Major steps in the journey
- Usually 4-8 phases
- From awareness to advocacy

**Actions**
- What the user does
- Specific behaviors
- Physical actions

**Touchpoints**
- Where user interacts
- Channels (website, app, phone, etc.)
- Different interactions

**Emotions/Feelings**
- Emotional highs and lows
- Satisfaction level
- Frustration points

**Pain Points**
- Where things go wrong
- Frustrations
- Unmet needs
- Obstacles

**Opportunities**
- Where you can improve
- Unmet needs you can fill
- Better ways to serve them

## Creating Journey Maps

### Step 1: Choose a Scenario

```
Pick ONE specific scenario:
- Who? (Which persona)
- What? (What goal/task)
- When? (What situation)
- Where? (What context)

Example:
Who: Sarah (Busy Manager)
What: Finding and booking a conference room
When: Last-minute, 15 minutes before meeting
Where: While walking to potential meeting spot
```

### Step 2: Research the Journey

**Methods:**
- Interview users
- Observe real usage
- Review analytics
- Ask customer support
- User testing
- Diary studies

**What to discover:**
- All steps they take
- Tools/channels they use
- What they think and feel
- Where they get stuck
- What they need but don't have

### Step 3: Map the Phases

```
Common Journey Phases:
1. Awareness - Problem recognition
2. Research - Exploring options
3. Consideration - Comparing solutions
4. Decision - Making choice
5. Purchase/Signup - Committing
6. Onboarding - Getting started
7. Usage - Regular use
8. Advocacy - Recommending to others

Choose relevant phases for your scenario
```

### Step 4: Define Actions

```
For each phase, list:
- What does user do?
- What are specific steps?
- What tools do they use?
- What information do they need?

Example (Booking Room):
Phase: Decision
Actions:
1. Opens calendar app
2. Checks availability
3. Searches for free room
4. Checks room features
5. Reserves room
```

### Step 5: Identify Touchpoints

```
Touchpoints = All interactions with your product

Examples:
- Website
- Mobile app
- Phone call
- Email
- Chat support
- In-person
- Social media
- Marketing emails

For each action, what touchpoint?
```

### Step 6: Add Emotions

```
Show emotional journey:
😭 Very frustrated
😞 Frustrated
😐 Neutral
😊 Satisfied
😄 Happy
😍 Delighted

Track how emotion changes throughout journey
```

### Step 7: Highlight Pain Points

```
Pain points = Where experience breaks down

Example:
Phase: Onboarding
Pain Point: "Too many fields to fill out"
Impact: High (users abandon)
Emotion: 😞 Frustrated

Mark pain points on map
```

### Step 8: List Opportunities

```
Opportunities = Where to improve

Formula:
Pain Point + Solution = Opportunity

Example:
Pain Point: "Too many fields"
Solution: "Use smart defaults"
Opportunity: "Auto-fill non-critical fields based on profile"
```

## Analyzing Journeys

### Finding Patterns

```
Look for:
1. Where emotions drop → Pain points
2. Where steps increase → Complexity
3. Where touchpoints change → Friction
4. Where users get stuck → Obstacles
5. Where needs aren't met → Opportunities
```

### Prioritizing Improvements

```
Impact x Effort Matrix:

High Impact    │ [DO THESE!]  │ Consider
Low Effort     │ Quick wins   │ Future
───────────────┼──────────────┼─────────
High Impact    │ Strategic    │ Difficult
High Effort    │ Priorities   │ Low ROI

Focus on: High impact + Low effort first
```

### Example Analysis

```
PAIN POINT: Email confirmation slow
- Impact: Users abandon if too slow
- Effort: Easy to fix (send SMS instead)
- Priority: HIGH (do immediately)

OPPORTUNITY: Smart defaults
- Impact: Reduces form-filling friction
- Effort: Medium (requires data)
- Priority: MEDIUM (plan for next sprint)
```

## Practical Exercises

### Exercise 1: Create Simple Journey Map
1. Choose one persona (from personas exercise)
2. Define their specific goal
3. Map 5-7 journey phases
4. List actions in each phase
5. Identify touchpoints
6. Add emotion ratings

### Exercise 2: Research a Journey
1. Conduct 2 user interviews
2. Ask them to walk through scenario
3. Take detailed notes
4. Capture their language
5. Note emotions and struggles
6. Create journey based on research

### Exercise 3: Create Detailed Journey Map
1. Take journey from Exercise 2
2. Add all 7 components:
   - Actions
   - Touchpoints
   - Emotions
   - Pain points
   - Opportunities
   - Current state
   - Desired state
3. Visualize in tool (Figma, Miro, etc.)

### Exercise 4: Identify Pain Points
1. Review journey map
2. Find 5 major pain points
3. Rate by impact and effort
4. Prioritize top 3
5. Design solutions for each
6. Document improvements

### Exercise 5: Share & Get Feedback
1. Create journey map poster
2. Share with team/colleagues
3. Gather feedback
4. Refine based on input
5. Get agreement from team
6. Use to drive product decisions

## Types of User Journeys

### Day-in-the-Life
**What**: Complete day from morning to evening
**Use**: Understanding broader context
**Best**: Revealing unexpected use cases

### Task Journey
**What**: Single focused task (e.g., checkout)
**Use**: Detailed optimization
**Best**: Improving specific flows

### Emotional Journey
**What**: Focusing on emotional ups and downs
**Use**: Creating memorable experiences
**Best**: Brand and retention

### Multi-channel Journey
**What**: Across different devices/channels
**Use**: Omnichannel experience
**Best**: Understanding channel switching

### Customer Lifecycle
**What**: Progression from new to loyal customer
**Use**: Long-term relationship
**Best**: Retention and growth strategies

## Common Mistakes

❌ **Too many phases** - Simplify to 5-7 key phases
❌ **Ignoring research** - Based on assumptions
❌ **Only happy path** - Show realistic journey including struggles
❌ **Too generic** - Should be for specific persona/scenario
❌ **Created once, never used** - Update as you learn
❌ **Beautiful but unused** - Make sure team references it
❌ **All customer perspectives same** - Different personas, different journeys

## Key Takeaways

✅ **Map specific scenarios** - Not generic journeys
✅ **Base on real research** - Talk to actual users
✅ **Show emotional journey** - Not just steps
✅ **Highlight pain points** - Where experience fails
✅ **Identify opportunities** - Where to improve
✅ **Share with team** - Drive decisions together
✅ **Update regularly** - As you learn more
